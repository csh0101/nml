# 卷01: 从零基础理解 NML

## 1. 你在做什么
你不是在“写一个 XML 文件”，你是在定义一套可执行写作系统。这个系统要同时满足两类对象:
- 人类作者可以看懂、维护、协作。
- Agent 可以稳定解析、执行、验收。

这就是 NML 的定位: 写作生产协议，而不是普通提纲。

## 2. 为什么不用纯自然语言提纲
纯自然语言提纲最大的问题不是表达能力，而是执行不稳定。

同样一句“这一场压抑一点、快一点、结尾留悬念”，不同模型、不同轮次会得到不同实现。你很难知道“偏差来自哪里”，也无法自动修正。

NML 通过标签把模糊意图分解为:
- 可识别结构（章节、场景、段落）
- 可校验引用（角色、地点）
- 可执行约束（句长、配比、禁止项）
- 可验收条件（硬性失败）

## 3. NML 的三层模型
### 3.1 人话层
作者用自然语言描述目标，例如:
- 这章要写主角绝境反击。
- 对话要少废话。
- 不要提前解释世界观。

### 3.2 规则层
NML 把目标转成标签和属性，例如:
- `章节蓝图` 定义章目标与钩子
- `配比` 定义描写/动作/对话比例
- `信息揭示` 定义必须提及和禁止提及

### 3.3 执行层
Agent 严格按标签输出文本，校验器按标签判定通过或失败。

## 4. 初学者最重要的认知
### 4.1 NML 不替代创意
它不负责“你想写什么故事”，只负责“让故事稳定产出”。

### 4.2 NML 的价值是可复现
在相同输入下，输出风格和结构波动更小，便于持续打磨。

### 4.3 NML 本质是契约
你和 Agent 的契约，不是建议。契约里写了就要执行，没写就不保证。

## 5. 新手起步只盯 4 个标签
### 5.1 `章节蓝图`
回答“本章完成什么”。没有它，整章会散。

### 5.2 `场景`
把章节拆成最小执行单元。Agent 一次只跑一个场景，才可控。

### 5.3 `任务`
回答“这个场景的核心动作是什么”。防止写偏。

### 5.4 `信息揭示`
回答“必须说什么、不能说什么”。防漏和防剧透。

## 6. 为什么这套方法值得坚持
- 它把“感觉”转成“参数”。
- 它把“争论”转成“验收条件”。
- 它把“返工”转成“定位问题后重跑”。

当你以后有多人协作、长连载、批量章节生成时，这些优势会非常明显。

## 7. 本卷小结
对零基础来说，NML 最难的不是标签，而是思维转换:
- 从“写得像”转为“可执行、可校验、可复用”。

只要你接受这个转换，后续每个标签都会非常自然。
