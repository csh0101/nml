# 卷02: 风格代词与参数模板

## 1. 为什么“代词”不够
你说“像某作家那样写”，对人类编辑有参考意义，但对 Agent 不够可执行。原因是:
- 代词是抽象的，模型会各自理解。
- 不同模型对同一代词的实现差异很大。
- 无法自动验收“到底像不像”。

所以代词只能做入口，不能做规则本体。

## 2. 正确做法: 代词 -> 参数模板
把 `冷峻武侠A` 展开为机器可执行字段:
- 句法: 句均字数、单句上限、长句占比
- 词汇: 形容词/副词密度、禁用词
- 推进: 段级信息点覆盖（由 `段/信息点` 承载）
- 对话: 台词句长、废话率
- 收束: 段尾动作化

这样你就能做到:
- 可执行
- 可比较
- 可回归测试

## 3. 两种写法的差别
### 3.1 失败写法
`文风="冷峻"`

问题: 太抽象，Agent 会加很多“看起来冷峻”的修辞，常常变得拖沓。

### 3.2 可执行写法
在风格模板中明确:
- `单句上限="24"`
- `形容词占比上限="10%"`
- 在段级结构中要求: 每个 `<段>` 至少 1 条 `<信息点>`

结果: 输出更短、更硬、更有推进。

## 4. 你关心的“简练不拖沓”怎么落地
核心不是“少写字”，而是“减少无效句”。

建议固定 4 个硬约束:
- 单句上限
- 长句占比上限
- 每段至少一个信息点（`<段>/<信息点>`）
- 禁用词表

这 4 个先跑通，再加其他项。

## 5. 参数不是越多越好
新手常见错误是把模板写成几十条，导致执行冲突。

建议顺序:
1. 句法
2. 推进
3. 词汇
4. 对话
5. 收束

每一步只加 1-2 条，观察输出，再微调。

## 6. 本卷小结
- 代词负责沟通，不负责执行。
- 参数模板才是执行体。
- 你的系统要追求“可校验的稳定输出”，不是“一次写得像”。
