# 卷07: 属性值标准化

## 1. 为什么要标准化属性值
同一个意思如果有多种写法，解析器就会出现歧义，Agent 执行也会漂移。

例如 `8%`、`0.08`、`百分之八` 三种写法对人类都懂，但机器会乱。

所以要统一语法，只允许一种或少量受控格式。

## 2. 建议支持的基础类型
- `int`
- `percent`
- `range`
- `enum`
- `csv<string>`
- `bool`
- `string`

所有标签属性都应映射到这几类之一。

## 3. 各类型推荐语法
### 3.1 int
- 合法: `24`, `3000`
- 非法: `24.5`, `二十四`

### 3.2 percent
建议统一为“带 `%` 的整数百分比”。
- 合法: `8%`, `35%`
- 非法: `0.08`, `8`, `百分之八`

### 3.3 range
建议统一为 `min-max`，且两端是整数。
- 合法: `10-16`
- 非法: `10~16`, `10 到 16`

### 3.4 enum
必须来自预定义枚举集合。
- 示例: `节奏="慢|中|快"` 中仅允许其一。
- 非法值直接报错，不做猜测修正。

### 3.5 csv<string>
用英文逗号分隔，去掉首尾空格。
- 合法: `压抑,肃杀,冷硬`
- 不推荐: 中文逗号 `，`（可在预处理阶段转换）

### 3.6 bool
只允许 `true|false`。
- 不接受: `是|否`、`1|0`（除非你明确做兼容层）

## 4. 解析与归一化策略
建议两步:
1. `parse`：按类型检查是否合法。
2. `normalize`：转成统一内部表示。

示例:
- `8%` -> `0.08`
- `10-16` -> `{min:10,max:16}`
- `压抑,肃杀` -> `["压抑","肃杀"]`

注意: 归一化是内部行为，对外仍保留原始书写以便审阅。

## 5. 容错策略建议
- 结构性错误（如 enum 非法值）: 直接失败。
- 轻微格式问题（如 csv 使用中文逗号）: 可自动修复并给 warning。
- 语义冲突（min>max）: 直接失败。

不要过度容错，否则坏数据会悄悄进入生产流程。

## 6. 推荐错误码
- `E1201 InvalidInteger`
- `E1202 InvalidPercent`
- `E1203 InvalidRangeFormat`
- `E1204 InvalidEnumValue`
- `E1205 InvalidBoolean`
- `E1206 InvalidCsv`
- `E1207 RangeSemanticError`

## 7. 属性类型映射建议（关键标签）
- `章节蓝图.字数目标`: int
- `章节蓝图.容差`: percent
- `句法.句均字数`: range
- `句法.单句上限`: int
- `句法.长句占比上限`: percent
- `场景.节奏`: enum
- `场景.氛围`: csv<string>
- `收束.段尾动作化`: bool

## 8. 新手最容易犯的 3 个错误
1. 同一个字段混用 `8%` 和 `0.08`。
2. enum 里放自由文本。
3. 用中文数字和自然语言表达数值。

你的规范要先禁止这些写法，再谈高级能力。

## 9. 本卷小结
属性值标准化不是小事，而是“可解析、可校验、可回归”的前提。

标准化做得越早，后面的解析器和执行器越省心。
