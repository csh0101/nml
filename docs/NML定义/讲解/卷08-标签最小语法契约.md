# 卷08: 标签最小语法契约

## 1. 本卷目的
前几卷讲了理念与流程，这一卷给出“可直接实现”的最小语法契约。

你可以把它当成:
- 解析器的输入约束
- 校验器的规则清单
- 文档写作者的统一标准

## 2. 记号说明
- `1`：必须出现一次
- `0..1`：可选，最多一次
- `0..n`：可选，可多次
- `1..n`：至少一次

## 3. 根层契约
### 3.1 小说项目
- 父: 无
- 子: `元数据(1)`, `设定集(0..1)`, `正文(0..1)`, `验收规则(0..1)`
- 属性: `名称(string,必填)`, `版本(string,必填)`

约束:
- 全文仅允许一个 `小说项目`。
- 若存在 `正文`，建议同时存在 `设定集`。

## 4. 元数据层契约
### 4.1 元数据
- 父: 小说项目
- 子: `写作准则(0..1)`, `风格库(0..1)`
- 属性: 无

### 4.2 写作准则
- 父: 元数据
- 子: 无
- 属性:
  - `文风(string,0..1)`
  - `视角(enum,0..1)`，取值: `第一人称|第三人称限定`
  - `细节度(int[1,5],0..1)`

### 4.3 风格库
- 父: 元数据
- 子: `风格模板(1..n)`
- 属性: 无

### 4.4 风格模板
- 父: 风格库
- 子: `句法(0..1)`, `词汇(0..1)`, `意象(0..1)`, `推进(0..1)`, `对话(0..1)`, `收束(0..1)`
- 属性:
  - `id(string,必填,全局唯一)`
  - `代词(string,0..1)`

## 5. 风格子标签契约
### 5.1 句法
- 父: 风格模板
- 子: 无
- 属性:
  - `句均字数(range,0..1)` 例 `10-16`
  - `单句上限(int,0..1)`
  - `长句占比上限(percent,0..1)`
  - `每段并列句上限(int,0..1)`

### 5.2 词汇
- 父: 风格模板
- 子: 无
- 属性:
  - `形容词占比上限(percent,0..1)`
  - `副词占比上限(percent,0..1)`
  - `抽象词占比上限(percent,0..1)`
  - `禁用词(csv<string>,0..1)`

### 5.3 意象
- 父: 风格模板
- 子: 无
- 属性:
  - `重复窗口(int,0..1)`
  - `同类意象重复上限(int,0..1)`

### 5.4 推进
- 父: 风格模板
- 子: 无
- 属性:
  - `每段必须新增信息点(int,0..1)`（历史写法，当前建议改由 `段/信息点` 结构约束表达）
  - `解释段上限(percent,0..1)`
  - `抒情段上限(percent,0..1)`

### 5.5 对话
- 父: 风格模板
- 子: 无
- 属性:
  - `台词句长上限(int,0..1)`
  - `解释性台词上限(percent,0..1)`
  - `废话率上限(percent,0..1)`

### 5.6 收束
- 父: 风格模板
- 子: 无
- 属性:
  - `段尾动作化(bool,0..1)`
  - `禁止空泛感叹(bool,0..1)`

## 6. 设定层契约
### 6.1 设定集
- 父: 小说项目
- 子: `角色(0..n)`, `地点(0..n)`
- 属性: 无

### 6.2 角色
- 父: 设定集
- 子: 无
- 属性:
  - `id(string,必填,全局唯一)`
  - `姓名(string,必填)`

### 6.3 地点
- 父: 设定集
- 子: 无
- 属性:
  - `id(string,必填,全局唯一)`
  - `名称(string,必填)`

## 7. 正文层契约
### 7.1 正文
- 父: 小说项目
- 子: `章节(1..n)`
- 属性: 无

### 7.2 章节
- 父: 正文
- 子: `章节蓝图(0..1)`, `场景(1..n)`
- 属性:
  - `序号(int,必填)`
  - `标题(string,必填)`
  - `风格模板引用(string,0..1,必须命中风格模板.id)`

### 7.3 章节蓝图
- 父: 章节
- 子: 无
- 属性:
  - `字数目标(int,必填)`
  - `容差(percent,0..1)`
  - `章节目标(string,必填)`
  - `章节钩子(string,0..1)`

### 7.4 场景
- 父: 章节
- 子: `任务(1)`, `配比(0..1)`, `信息揭示(0..1)`, `段落计划(0..1)`, `场景收束(0..1)`
- 属性:
  - `id(string,必填,章节内唯一)`
  - `地点引用(string,必填,命中地点.id)`
  - `字数目标(int,0..1)`
  - `节奏(enum,0..1)`，取值建议: `慢|中|快|慢转中|中转快`
  - `氛围(csv<string>,0..1)`
  - `视角角色(string,0..1,建议命中角色.id)`

### 7.5 任务
- 父: 场景
- 子: 无
- 属性:
  - `目标(string,必填)`

### 7.6 配比
- 父: 场景
- 子: 无
- 属性:
  - `描写(int,0..1)`
  - `对话(int,0..1)`
  - `动作(int,0..1)`
  - `心理(int,0..1)`
  - `校验(string,0..1)` 建议值: `和为100`

### 7.7 信息揭示
- 父: 场景
- 子: `必须提及(0..n)`, `禁止提及(0..n)`
- 属性: 无

### 7.8 必须提及
- 父: 信息揭示
- 子: 无
- 属性: 无
- 文本: 必填，非空字符串

### 7.9 禁止提及
- 父: 信息揭示
- 子: 无
- 属性: 无
- 文本: 必填，非空字符串

### 7.10 段落计划
- 父: 场景
- 子: `段(1..n)`
- 属性:
  - `校验(enum,0..1)`，取值: `和等于场景字数目标|对齐场景配比`
  - `偏差上限(percent,0..1)`，仅 `校验=对齐场景配比` 时生效
  - `流转校验(enum,0..1)`，取值: `开启`
  - `状态校验(enum,0..1)`，取值: `开启`
  - `体力演化(enum,0..1)`，取值: `同级或下降`
  - `情绪演化(enum,0..1)`，取值: `受限迁移`
  - `关系校验(enum,0..1)`，取值: `开启`
  - `关系演化(enum,0..1)`，取值: `波动不超过1`
  - `关系类型演化(enum,0..1)`，取值: `严格一致|受限迁移`

### 7.11 段
- 父: 段落计划
- 子: `信息点(1..n)`, `状态(0..n)`, `关系状态(0..n)`
- 属性:
  - `序号(int,0..1,段落计划内唯一)`
  - `字数目标(int,必填,正整数)`
  - `功能(enum,必填)`，取值: `铺垫|推进|转折|爆发|回落|钩子`
  - `焦点(enum,必填)`，取值: `环境|动作|对话|心理|信息`
  - `节奏(enum,必填)`，取值: `慢|中|快`
  - `视角(string,0..1,建议命中角色.id)`
  - `对话占比(int,0..1)`，当 `段落计划.校验=对齐场景配比` 时必填，范围 `0-100`
  - `动作占比(int,0..1)`，当 `段落计划.校验=对齐场景配比` 时必填，范围 `0-100`

### 7.12 场景收束
- 父: 场景
- 子: 无
- 属性:
  - `类型(enum,必填)`，取值: `转折|悬念|钩子|回落`
  - `落点(string,必填)`

### 7.13 信息点
- 父: 段
- 子: 无
- 属性:
  - `类型(enum,必填)`，取值: `伏笔|反转|设定|情绪|冲突|行动`
  - `权重(int,必填)`，范围 `1-5`
- 文本: 必填，非空字符串

### 7.14 状态
- 父: 段
- 子: 无
- 属性:
  - `角色(string,必填,建议命中角色.id)`
  - `阶段(enum,必填)`，取值: `入|出`
  - `体力(enum,0..1)`，取值: `极低|低|中|高`
  - `情绪(enum,0..1)`，取值: `平静|克制|紧绷|决绝|麻木`

### 7.15 关系状态
- 父: 段
- 子: 无
- 属性:
  - `角色(string,必填,建议命中角色.id)`
  - `目标(string,必填,建议命中角色.id)`
  - `阶段(enum,必填)`，取值: `入|出`
  - `强度(int,必填)`，范围 `1-5`
  - `类型(enum,0..1)`，取值: `敌对|对抗|戒备|试探|中立|合作|信任|依赖|背叛`

## 8. 验收层契约
### 8.1 验收规则
- 父: 小说项目
- 子: `硬性失败(0..n)`
- 属性: 无

### 8.2 硬性失败
- 父: 验收规则
- 子: 无
- 属性:
  - `条件(string,必填)`

## 9. 顺序约束建议
在同一父标签内，建议固定顺序:
1. 配置类（如章节蓝图）
2. 执行类（场景/任务/段落计划）
3. 验收类（场景收束/验收规则）

固定顺序不是必须，但会显著降低人工维护成本。

## 10. 本卷小结
这一卷给出的就是你可直接实现的“最小语法合同”。

只要解析器按本合同校验，文档作者、Agent、验收器三方就能统一口径。
