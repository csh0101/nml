# 卷16: NML-Standard-v2.1 主文档草案

## 1. 本卷目标
把前 15 卷内容收敛为可发布标准的单文件骨架。

你可以把这一卷视为“发布前检查单 + 主文档生成指南”。

## 2. 主文档建议路径
建议固定主标准文件:
- `docs/NML定义/NML-Standard-v2.1.md`

该文件是唯一权威入口，其他分卷作为解释与附录来源。

## 3. 主文档必须包含的 8 章
1. 范围与目标
2. 术语与记号
3. 标签语法契约
4. 语义优先级与冲突裁决
5. 执行流程
6. 验收规则与错误码
7. 版本与迁移
8. 示例与附录索引

缺少任一章节都会导致落地实现出现口径歧义。

## 4. 发布前最小检查
- 是否存在条款编号（NML-XXX-YYY）
- 是否区分 MUST/SHOULD/MAY
- 是否有可运行最小示例
- 是否有错误码与迁移规则
- 是否有版本声明要求

## 5. 评审流程建议
1. 架构评审（标签与流程）
2. 实现评审（解析器是否能执行条款）
3. 样例评审（happy/failure/edge）
4. 冻结并发布 `v2.1`

## 6. 本卷小结
你现在已经具备发布标准的全部材料。关键不是“继续扩写”，而是“冻结核心、启动实现、跑回归”。

下一步建议: 以 `NML-Standard-v2.1.md` 为唯一输入，启动解析器 MVP。
