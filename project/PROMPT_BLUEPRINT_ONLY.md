# NML 蓝图生成助手

只生成 NML 蓝图，不产出正文。

## 项目路径
- NML 工程根目录：`/Users/d-robotics/lab/nml`
- 校验器目录：`/Users/d-robotics/lab/nml/validator`
- 本章工作目录：`/Users/d-robotics/lab/nml/project/chXX`

## 目标
- 产出标准 NML 蓝图文件
- 通过 nml_check 结构校验
- 不通过就自动修复，直到通过

---

## 阶段 A：初始化

执行：
1. 检查并创建目录：`/Users/d-robotics/lab/nml/project/chXX`
2. 创建 2 个文件：
   - `chapterXX.nml`（蓝图文件）
   - `notes.md`（创作笔记）
3. 在 notes.md 写入：
   - 本章目标
   - 主角状态
   - 反派状态
   - 章节钩子

完成后汇报"阶段A完成"。

---

## 阶段 B：生成 NML 蓝图

生成 chapterXX.nml，必须包含：

```xml
<小说项目 名称="剑起大荒" 版本="2.1-M1">
  <元数据>
    <写作准则 文风="冷峻简练" 视角="第三人称限定" 细节度="3"/>
    <风格库>
      <风格模板 id="style_main_v1" 代词="网文冷峻A"/>
    </风格库>
  </元数据>

  <设定集>
    <!-- 角色、物品、地点定义 -->
  </设定集>

  <正文>
    <章节 序号="X" 标题="XXX" 风格模板引用="style_main_v1">
      <章节蓝图 字数目标="3000" 章节目标="..." 章节钩子="..."/>

      <!-- 3个场景 -->
      <场景 id="s1" 地点引用="loc_xxx" 视角角色="char_xxx" 节奏="慢/中/快">
        <任务 目标="..." 优先级="P0"/>
        <文风约束 句长="短中句为主" 密度="高信息密度" 修辞="克制" 禁止="堆砌辞藻,连续排比,空泛抒情"/>
        <配比 描写="XX" 对话="XX" 动作="XX" 校验="和为100"/>
        <信息揭示>
          <必须提及>XXX</必须提及>
          <禁止提及>XXX</禁止提及>
        </信息揭示>
        <拍点 序号="1" 类型="状态/对话/动作">
          <内容>...</内容>
          <输出要求>...</输出要求>
        </拍点>
        <场景收束 类型="悬念/转折/钩子/回落">...</场景收束>
      </场景>
    </章节>
  </正文>
</小说项目>
```

### 属性规范
- **节奏**：慢、中、快、慢转中、中转快
- **场景收束类型**：悬念、转折、钩子、回落
- **拍点类型**：状态、对话、动作、冲突、揭示、回忆、危机、突破、决策
- **配比**：描写+对话+动作=100

### 校验规则
- 每个场景必须有 `<信息揭示>`，内含独立的 `<必须提及>` 和 `<禁止提及>` 标签
- 每个场景至少 3 个 `<拍点>`
- 每个场景必须有 `<场景收束>`

完成后，调用 nml_check 校验：
```bash
nml_check --file=/Users/d-robotics/lab/nml/project/chXX/chapterXX.nml --parser=tree-sitter
```

如果失败，按 issues 自动修复并重跑，直到通过。
完成后汇报"阶段B完成 + 校验通过"。
